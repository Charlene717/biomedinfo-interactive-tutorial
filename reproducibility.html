概覽

重現性危機與詞彙

2016 年 Nature 對 1,576 名科學家的調查：70% 曾無法重現他人實驗，超過 50% 無法重現自己的研究。Baker (2016)、Ioannidis (2005) 與 Open Science Collaboration (2015) 都點出生醫研究的重現性問題。基本詞彙：

  • Computational reproducibility（計算重現）：同樣資料 + 同樣程式碼 → 同樣結果。最基本要求。
  • Replicability（複製）：新資料 + 同方法 → 相似結論。
  • Robustness（穩健）：稍微不同的資料/方法 → 相似結論。
  • Generalizability（推論性）：不同族群/情境 → 相似結論。

Nature's 2016 survey of 1,576 scientists: 70% had failed to reproduce others' experiments; over 50% couldn't reproduce their own. Baker (2016), Ioannidis (2005), and the Open Science Collaboration (2015) all highlight a reproducibility problem in biomedicine. Basic vocabulary:

  • Computational reproducibility: same data + same code → same result. The minimum bar.
  • Replicability: new data + same method → similar conclusion.
  • Robustness: slightly varied data/method → similar conclusion.
  • Generalizability: different populations/contexts → similar conclusion.
四層工具

一、四層工具

🌿 版本控制

Git + GitHub/GitLab：追蹤每行程式碼變動。main 為穩定、dev 為開發、PR 為審查機制。
論文版本：用 tag (例 v1.0-submission) 凍結特定版本。

Git + GitHub/GitLab: track every line change. main = stable, dev = development, PRs for review.
Manuscript versions: use tags (e.g., v1.0-submission) to freeze specific states.

📦 環境管理

conda / mamba（最常用）、renv（R 專案鎖定）、poetry / uv（現代 Python）。每個專案一個獨立環境 + 鎖定檔（environment.yml、renv.lock、pyproject.toml）。

conda / mamba (most common), renv (R project pinning), poetry / uv (modern Python). One environment per project + lockfile (environment.yml, renv.lock, pyproject.toml).

🐳 容器化

Docker / Singularity / Apptainer：把 OS 函式庫 + 工具 + 環境全打包成一個映像檔。HPC 通常用 Singularity（不需 root）；DockerHub / Biocontainers 提供現成映像。

Docker / Singularity / Apptainer: bundle OS libs + tools + environment into one image. HPC typically uses Singularity (no root needed). DockerHub / Biocontainers provide ready-made images.

🔄 工作流引擎

Snakemake（Python，DAG 自動）、Nextflow（DSL，跨平台佳）、WDL / Cromwell（雲端流程，Terra/AnVIL）。從 raw FASTQ 到結果一鍵跑完。

Snakemake (Python, auto-DAG), Nextflow (DSL, cross-platform), WDL / Cromwell (cloud-native, Terra/AnVIL). Run from raw FASTQ to result in one command.

動態報告

二、動態文件

把程式碼、輸出、文字寫在同一份檔案，並可一鍵 render 成 HTML / PDF / DOCX。優點：

  • 文字與程式碼緊密綁定——修改資料後重 render，圖表自動更新
  • literate programming——讀者可以看到每段邏輯
  • 支援 ggplot、Plotly、LaTeX、mermaid 等

2025 年最推薦：Quarto（取代 R Markdown、Jupyter book，可同時支援 R / Python / Julia / Observable）。

Embed code, output, and prose in one file; render to HTML / PDF / DOCX with one command. Pros:

  • Tight coupling between text and code — figures auto-update when data changes
  • Literate programming — readers follow every logical step
  • Supports ggplot, Plotly, LaTeX, mermaid, etc.

2025 top pick: Quarto (replaces R Markdown / Jupyter Book; supports R / Python / Julia / Observable in one file).

# Minimal RNA-seq Snakemake example
SAMPLES = ["S1", "S2", "S3"]

rule all:
    input:
        "results/de_results.tsv"

rule fastqc:
    input:  "data/raw/{sample}.fq.gz"
    output: "results/qc/{sample}_fastqc.html"
    shell:  "fastqc {input} -o results/qc"

rule align:
    input:  fq="data/raw/{sample}.fq.gz", idx="data/ref/star_idx"
    output: "results/bam/{sample}.bam"
    threads: 8
    shell:  "STAR --readFilesIn {input.fq} --genomeDir {input.idx} "
            "--runThreadN {threads} --outFileNamePrefix results/bam/{wildcards.sample}_"

rule deseq2:
    input:  expand("results/bam/{s}.bam", s=SAMPLES)
    output: "results/de_results.tsv"
    script: "src/02_deseq2.R"
# Reproducible bioinformatics image

📝 隨堂測驗

Q1. Computational reproducibility 的最基本定義？

A. 不同樣本能得到相同結果Same result on a new sample
B. 同樣資料+同樣程式碼能得到同樣結果Same data + same code → same result
C. 不同方法得到相同結論Different methods → same conclusion
D. 不同實驗室得到相同結果Different labs → same result
這是重現性的最低門檻：任何人取得你的資料和程式碼都能得到同樣輸出。This is the floor: anyone with your data and code reproduces your output.

Q2. HPC 常用哪種容器？

A. Docker
B. Singularity / Apptainer
C. VMware
D. 不用容器No containers
HPC 通常不允許 root 權限。Singularity/Apptainer 是不需 root 的替代方案，且能直接讀 Docker images。HPC clusters disallow root. Singularity/Apptainer is the rootless alternative and can pull Docker images.

Q3. 處理過的資料放哪？

A. 私人電腦Local hard drive
B. Zenodo / FigShare 等公開資料庫並取得 DOIPublic repository like Zenodo / FigShare with a DOI
C. 補充材料 PDF 附錄PDF supplement
D. 完全不公開Don't share
Zenodo/FigShare 提供永久 DOI、版本控管與 metadata 範本，符合 FAIR 原則。Zenodo/FigShare provide persistent DOIs, versioning, and metadata templates — meeting the FAIR principles.

Q4. raw 與 processed 資料為何要分開？

A. 節省硬碟Save disk space
B. raw 應視為唯讀，processed 可隨時重建Raw is read-only; processed can be regenerated
C. 符合 GDPRRequired by GDPR
D. 提升 I/O 效能Boost I/O performance
raw 必須維持原貌（萬一處理錯誤可回到起點）。processed 是程式碼 + raw 的衍生物，可隨時重建。Raw must be untouched (so you can re-run from scratch). Processed is a derivative of code + raw — regeneratable any time.
