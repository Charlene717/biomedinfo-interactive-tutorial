公開資料庫探索

從 NCBI、Ensembl 到 TCGA、UK Biobank——學會找到、下載並正確引用公開的生醫資料。

From NCBI and Ensembl to TCGA and UK Biobank — learn to find, download, and correctly cite public biomedical data.

概覽

為什麼公開資料庫是核心？

生醫資料庫的爆炸式成長使得「不需自己做實驗就能做出新發現」變得可能。一個小型實驗室可以用公開的 TCGA 泛癌資料做出 Nature 等級的研究。但這也帶來新挑戰：

  • 各資料庫的命名規則、版本與註釋方式差異極大
  • 原始 cohort 的納入排除標準會影響結論可推論性
  • 批次效應、缺失值與資料格式陷阱無處不在
  • 使用前需確認授權條款（如 dbGaP、UK Biobank 須申請）

The explosion of biomedical databases makes Nature-level discoveries possible without running a single wet-lab experiment. A small lab can publish from TCGA pan-cancer data alone. But this comes with new challenges:

  • Databases differ in nomenclature, versions, and annotation schemes
  • The original inclusion/exclusion criteria shape what your conclusions can generalize to
  • Batch effects, missing values, and format pitfalls are everywhere
  • Licensing matters (dbGaP, UK Biobank require formal application)
⚠️
引用是責任：使用 TCGA 必須 cite TCGA Research Network；使用 UK Biobank 要寫 application ID；使用 ClinVar 要記錄存取日期。讓資料生產者得到應有的學術 credit。 Citing is a duty: TCGA users must cite the TCGA Research Network; UK Biobank users must include the application ID; ClinVar users must log the access date. Give data producers their due credit.
基因/序列

一、基因組與序列資料庫

🧬 NCBI (National Center for Biotechnology Information)

美國 NIH 維護的綜合性資料庫平台。主要工具：GenBank（序列）、RefSeq（標準參考序列）、dbSNP（單核苷酸變異）、ClinVar（臨床相關變異）、GEO（基因表現微陣列與 NGS 資料）、SRA（原始定序 reads）、OMIM（孟德爾遺傳病）。

The NIH umbrella platform. Key tools: GenBank (sequences), RefSeq (curated reference), dbSNP (SNVs), ClinVar (clinically annotated variants), GEO (expression arrays & NGS), SRA (raw reads), OMIM (Mendelian diseases).

🌍 Ensembl / Ensembl Genomes

EMBL-EBI 維護，以「基因組瀏覽器 + REST API」為特色。提供轉錄本層級的註釋、跨物種比對、調控元素 (Regulatory Build)。版本化清楚（如 Ensembl 110, 111）——這對可重現性極重要。

EMBL-EBI's genome browser plus REST API. Transcript-level annotation, cross-species alignments, regulatory build. Clear versioning (e.g., Ensembl 110, 111) — vital for reproducibility.

🔭 UCSC Genome Browser

視覺化基因組首選，BLAT 快速比對、Table Browser 批次下載 BED/BigWig。Track Hub 機制讓你能載入自己的資料對齊參考。

The top genome visualization tool. BLAT for fast alignment, Table Browser for bulk BED/BigWig downloads. Track Hubs let you overlay your own data on the reference.

🧪 EMBL-EBI 其他工具 / Other EMBL-EBI tools

UniProt（蛋白質）、InterPro（蛋白質家族與功能域）、PRIDE（質譜資料）、ChEMBL（生物活性化合物）、BioModels（系統生物學模型）。

UniProt (proteins), InterPro (domains/families), PRIDE (proteomics), ChEMBL (bioactive compounds), BioModels (systems biology).

三、程式化存取資料庫

# Download TCGA RNA-seq + clinical via TCGAbiolinks
library(TCGAbiolinks)

query <- GDCquery(
  project="TCGA-LUAD",
  data.category="Transcriptome Profiling",
  data.type="Gene Expression Quantification",
  workflow.type="STAR - Counts"
)
GDCdownload(query, directory="GDCdata")
luad <- GDCprepare(query)
# Pull paired clinical data
clin <- GDCquery_clinic("TCGA-LUAD", type="clinical")

# Query GEO via GEOquery
library(GEOquery)
gse <- getGEO("GSE68465", GSEMatrix=TRUE)
expr <- exprs(gse[[1]])
# GDC API via requests; UK Biobank via DNAnexus / RAP
import requests, pandas as pd

url = "https://api.gdc.cancer.gov/files"
filters = {
  "op":"and","content":[
    {"op":"in","content":{"field":"cases.project.project_id","value":["TCGA-LUAD"]}},
    {"op":"in","content":{"field":"data_type","value":["Gene Expression Quantification"]}}
  ]}
params = {"filters":str(filters),"format":"TSV","size":"10000"}
r = requests.get(url, params=params)
files = pd.read_csv(pd.compat.StringIO(r.text), sep="\t")

# Ensembl REST API — lookup gene
ens = requests.get(
  "https://rest.ensembl.org/lookup/symbol/homo_sapiens/EGFR",
  headers={"Content-Type":"application/json"}
).json()
print(ens["id"], ens["start"], ens["end"])
# GDC client (fastest for bulk TCGA)
gdc-client download -m manifest.txt -t token.txt

# SRA toolkit — get raw FASTQ
prefetch SRR123456
fasterq-dump SRR123456 --split-files --threads 8

# Aspera Connect — UK Biobank bulk download
ukbfetch -afoo.enc -bdata.bulk

# dbGaP — controlled access via SRA toolkit + prj token
prefetch --ngc prj_12345_D12345.ngc SRR987654
陷阱清單

四、常見陷阱與最佳實踐

🆔 ID 對應地獄

Gene Symbol、Ensembl ID、Entrez ID、Uniprot ID 互轉時，每個版本都對應不同。Excel 把 SEPT2、MARCH1 自動轉日期是經典案例（HGNC 已在 2020 重新命名）。請用 biomaRt、AnnotationHub、mygene 程式化轉換，並鎖定版本。

Mapping between Gene Symbols, Ensembl IDs, Entrez IDs, UniProt IDs is version-dependent. Excel auto-converting SEPT2/MARCH1 to dates is legendary (HGNC renamed them in 2020). Use biomaRt, AnnotationHub, or mygene and pin a version.

🧬 基因組組裝版本

GRCh37 (hg19) vs GRCh38 (hg20) 的座標不能直接相比。混用會讓 BED 檔讀錯位置。需用 UCSC liftOver 或 CrossMap 轉換。

GRCh37 (hg19) and GRCh38 (hg20) coordinates do NOT match. Mixing them silently corrupts BEDs. Use UCSC liftOver or CrossMap.

🔁 批次效應

同樣的 TCGA-LUAD 樣本，定序中心 (BCM vs Broad) 或 RNA-seq 批次會造成大型偏差。合併多 GEO 數據時務必用 ComBat、limma::removeBatchEffect 或將 batch 納入回歸模型。

Even within TCGA-LUAD, sequencing center (BCM vs Broad) or RNA-seq batch creates large bias. When merging GEO datasets always apply ComBat, limma::removeBatchEffect, or include batch as a covariate.

📋 樣本選擇偏倚

TCGA 多為手術切除標本 → 偏向早期、可切除腫瘤；MIMIC 來自單一 ICU → 病情嚴重。把結論推到「所有患者」之前先看 cohort table。

TCGA samples come from surgical resections → biased to early, resectable tumors. MIMIC is from one ICU → severe illness. Always read the cohort table before generalizing.
