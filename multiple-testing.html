多重檢定校正

同時做 20,000 個基因檢定？沒校正就會有上千個假陽性。FWER vs FDR 是現代組學分析的核心抉擇。

Testing 20,000 genes at once? Without correction you'd see thousands of false positives. FWER vs FDR is the core dilemma in modern omics.

概覽

為什麼一定要校正？

當你做 1 次檢定、α=0.05 → 至少 1 次假陽性的機率為 5%。
當你同時做 20,000 次（如 RNA-seq 所有基因）→ 期望出現 1,000 次假陽性，且至少 1 次假陽性的機率幾乎是 100%。沒有校正的「顯著基因列表」沒有意義。

One test at α=0.05 → 5% chance of one false positive.
20,000 simultaneous tests (e.g., RNA-seq) → 1,000 expected false positives, and ~100% chance of at least one. An uncorrected "significant gene list" is meaningless.

🚨
常見錯誤：把 RNA-seq 報表的 p < 0.05 直接畫熱圖 → 整張圖幾乎全是雜訊。報告應使用 padj / q-value（即校正後 p）。 Common mistake: Drawing a heatmap of all genes with raw p < 0.05 yields mostly noise. Report padj / q-value (corrected) instead.
兩種錯誤率

一、FWER vs FDR

— FWER FDR
全名 Family-Wise Error Rate False Discovery Rate
控制目標 至少 1 次假陽性的機率P(≥1 false positive) 假陽性佔顯著結果的比例期望值Expected fraction of false positives among rejections
代表方法 Bonferroni, Holm, Šidák Benjamini-Hochberg, Storey q-value
嚴格度 非常嚴格（保守）Very strict (conservative) 較寬鬆，保留更多發現More lenient, retains more discoveries
適用 GWAS、確證性試驗、少數但關鍵的檢定GWAS, confirmatory trials, few but crucial tests 組學探索（RNA-seq, ChIP-seq）、篩選候選Omics screening (RNA-seq, ChIP-seq), candidate selection
💡
記憶法：FWER = 「一個都不要錯」；FDR = 「大部分是真的就好」。RNA-seq 寧可放過 1,000 個真基因，也不要 50 個假陽性 → 用 FDR。GWAS 怕假陽性害大家追錯目標 → 用 FWER (Bonferroni)。 Mnemonic: FWER = "not even one false call"; FDR = "most of my hits should be real." RNA-seq prefers FDR (10% noise acceptable). GWAS prefers FWER (Bonferroni) — false leads are costly.
主要方法

二、五大方法

🔒 Bonferroni

最簡單最嚴格：α* = α / m，其中 m = 檢定次數。控制 FWER，但極保守，會大幅降低檢定力。基因組學若用會幾乎找不到任何顯著結果。

Simplest and strictest: α* = α / m, where m = number of tests. Controls FWER but very conservative — kills statistical power. Genome-wide use yields almost no hits.

📉 Holm-Bonferroni

Step-down 版 Bonferroni：將 p 值排序，第 i 小用 α/(m-i+1) 比較。比 Bonferroni 多一點檢定力，但仍控制 FWER。

Step-down Bonferroni: sort p-values; for the i-th smallest, compare to α/(m-i+1). Slightly more powerful, still FWER-controlling.

📊 Benjamini-Hochberg (BH)

1995 年提出，控制 FDR 的金標準。將 p 值排序，找最大的 i 使 p(i) ≤ i·α/m，拒絕前 i 個。簡單、強大、廣泛使用。RNA-seq DESeq2 / edgeR 預設方法。

Proposed in 1995. Gold-standard for FDR. Sort p-values; find largest i s.t. p(i) ≤ i·α/m, then reject the top i. Simple, powerful, ubiquitous. The default in DESeq2 / edgeR.

🌊 Storey q-value

2002 年提出，比 BH 進一步：估計「真實虛無比例 π₀」後再校正，當 H₀ 比例不接近 100% 時更靈敏。R 套件 qvalue。

Proposed in 2002. Extends BH by estimating the proportion of true nulls (π₀); more sensitive when not all H₀ are true. R package qvalue.

🧬 GWAS 通用 α=5×10⁻⁸

不算嚴格意義的「方法」，而是約定俗成的閾值：0.05 / 10⁶（人類基因組獨立 LD block 數）。等同於針對全基因組的 Bonferroni。

Not strictly a "method" — a community convention: 0.05 / 10⁶ (independent LD blocks). Effectively a genome-wide Bonferroni.

🎯 置換檢定

打亂 group label 重算統計量，建立 null distribution。常用於 GSEA、ChIP-seq peak calling。能處理檢定間相關性。

Shuffle group labels and recompute the statistic to build a null distribution. Used in GSEA, ChIP-seq peak calling. Handles correlated tests well.

程式碼

實作範例

# p.adjust supports: holm, hochberg, hommel, bonferroni, BH (=fdr), BY, none
pvals <- c(1e-7,0.001,0.02,0.04,0.05,0.6)
padj_bonf <- p.adjust(pvals, method="bonferroni")
padj_bh   <- p.adjust(pvals, method="BH")
data.frame(p=pvals, bonf=padj_bonf, BH=padj_bh)

# Storey q-value (estimates pi0)
library(qvalue)
q <- qvalue(pvals)
summary(q); plot(q)
import numpy as np
from statsmodels.stats.multitest import multipletests

pvals = np.array([1e-7,0.001,0.02,0.04,0.05,0.6])
# method: bonferroni, holm, hommel, fdr_bh, fdr_by, fdr_tsbh (two-stage)
rej, padj_bh, _, _ = multipletests(pvals, alpha=0.05, method="fdr_bh")
rej, padj_bonf,_,_ = multipletests(pvals, alpha=0.05, method="bonferroni")

for p, pb, pbh in zip(pvals, padj_bonf, padj_bh):
    print(f"p={p:.2e}  bonf={pb:.2e}  BH={pbh:.2e}")

📝 隨堂測驗

Q1. 10,000 個檢定，α=0.05，FP？

A. 50
B. 500
C. 5,000
D. 0.05
期望 FP ≈ m × α = 10,000 × 0.05 = 500。Expected FP ≈ m × α = 10,000 × 0.05 = 500.

Q2. RNA-seq DE 最常用校正？

A. Bonferroni
B. Benjamini-Hochberg (FDR)
C. Holm
D. 不校正No correction
RNA-seq 是「探索性」分析，研究者願意接受 10% 左右的假陽性以保留發現 → BH FDR。DESeq2/edgeR 預設即 BH。RNA-seq is exploratory — researchers tolerate ~10% false positives to retain discoveries → BH FDR. DESeq2/edgeR default to BH.

Q3. Bonferroni vs BH 的差別？

A. 兩者控制相同錯誤率Both control the same error rate
B. Bonferroni 控 FWER，BH 控 FDR；後者更靈敏Bonferroni controls FWER, BH controls FDR — BH is more sensitive
C. Bonferroni 更靈敏Bonferroni is more sensitive
D. 兩者完全等價Both are equivalent
Bonferroni 防的是「任何一個」假陽性（嚴格）；BH 容忍少量假陽性以保留更多真陽性（靈敏）。Bonferroni guards against any false positive (strict); BH tolerates a small fraction to retain more true discoveries (sensitive).

Q4. GWAS 5×10⁻⁸ 對應的校正是？

A. 針對 ~10⁶ 個獨立 LD block 的 BonferroniBonferroni for ~10⁶ independent LD blocks
B. Benjamini-Hochberg FDR
C. Storey q-value
D. 沒有校正No correction
人類基因組約有 10⁶ 個獨立 LD block，0.05/10⁶ = 5×10⁻⁸ 即族群尺度 Bonferroni 閾值。The human genome has ~10⁶ independent LD blocks; 0.05/10⁶ = 5×10⁻⁸ is the genome-wide Bonferroni threshold.
